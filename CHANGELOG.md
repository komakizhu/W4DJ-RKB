# W4DJ RKB 更新日志

## v2.2.0

### Added

- 每个任务的输入来源现在同时支持歌曲文件夹和单曲文件，可通过文件选择器或拖放添加 MP3、FLAC、NCM、WAV、AIFF。
- 新增输出文件冲突策略：已存在文件时跳过或覆盖；高级选项支持仅更新元数据。
- 新增文件名规则设置，可选择“标题 - 艺术家”“艺术家 - 标题”或保留原文件名；默认仍为“标题 - 艺术家”。
- 新增任务取消功能，可取消单个任务或全部运行中的任务；当前歌曲会安全结束后再停止。
- 新增未完成任务恢复机制。任务进度会持续保存，暂停、取消或应用意外关闭后，可从转换历史继续剩余文件。
- 新增磁盘空间检查。转换前显示目标磁盘可用空间，空间不足时会阻止任务开始并说明原因。
- 新增错误分类，区分文件损坏或无法读取、格式不支持、FFmpeg 失败、输出目录无权限、磁盘空间不足、文件名非法和其他错误。
- 新增历史记录管理，可删除单条历史或清空全部历史，不会影响已经生成的音频文件。
- 新增“关于”窗口，可查看应用版本、开发者和项目主页。

### Improved

- 自动清理跨平台不允许的文件名字符，处理 Windows 保留名称、末尾空格和句点，并限制过长文件名。
- “仅更新元数据”会直接更新现有 MP3、WAV 或 AIFF 输出，不重新编码音频；支持从 MP3、WAV、AIFF、FLAC 和 NCM 读取元数据。
- 转换历史会持续记录剩余项目与已完成数量，并支持在重新打开应用后继续任务。
- 两个并行任务写入历史记录时会串行保存，避免记录相互覆盖。
- 预检窗口现在同时显示预计输出大小和目标磁盘可用空间。

### Fixed

- 修复单曲路径会被预检和正式转换误判为“不是目录”的问题。
- 阻止输出目标与输入单曲相同时覆盖或修改原曲，并确保清理旧格式时不会删除位于输出目录中的源文件。
- 修复两个任务计划写入同一输出文件时可能重复转换的问题；重复项目现在只交由前一个任务处理。
- 修复取消或暂停后只能重新开始整批任务、无法准确继续未完成文件的问题。
- 修复覆盖和仅更新元数据策略在预检结果与实际目标文件之间可能不一致的问题。
- 修复“仅更新元数据”可能错误使用推算目标路径，而不是实际已存在文件路径的问题。
- 修复转换历史中的跳过数量可能被错误记录为全部已存在文件数量的问题。
- 修复两个任务同时结束时可能竞争写入历史文件、导致部分历史丢失的问题。

### Breaking Changes

- 无。v2.1.0 的偏好设置与历史记录会自动兼容；新增字段使用安全默认值。

## v2.1.0

### Added

- 新增转换前汇总确认窗口，一次查看新增、已存在、将跳过和错误文件数量。
- 新增预计输出大小，开始转换前即可了解大致磁盘空间需求。
- 新增转换历史记录，保存任务时间、文件数量、失败文件和输出目录。
- 新增失败任务重试，可只重试失败歌曲，并支持导出错误报告。

### Improved

- 确认窗口与实际转换使用同一份预检结果，避免确认后统计与实际执行不一致。
- 历史记录按任务集中展示，方便追踪多次转换结果。
- 输出模式和 WAV/AIFF 选择操作更稳定，快速操作时界面响应更连贯。
- 更新使用说明，补充预检确认、历史记录和失败重试的使用方式。

### Fixed

- 修复反复点击兼容模式时滑动动画不断重新播放的问题。
- 修复快速切换 WAV 和 AIFF 时按钮短暂卡顿或状态不同步的问题。
- 修复部分任务失败后无法明确查看失败文件的问题。

### Breaking Changes

- 无。

## v2.0.1

### Added

- 支持两个同步任务同时配置和执行，适合批量处理网易云音乐与 SoundCloud 曲库。
- 支持兼容模式输出 MP3，以及无损模式输出 WAV 或 AIFF。
- 支持网易云音乐 `.ncm` 解密转换，并支持常见音频文件处理。
- 支持保留歌曲元数据和封面，生成适合 DJ 软件和设备使用的音频文件。
- 提供中英文界面、浅色/深色主题和拖放导入等基础操作体验。
- 提供 Apple Silicon、Intel Mac 和 Windows 安装包。

### Improved

- 优化批量转换进度、任务状态和输出目录配置体验。
- macOS 安装包使用易懂的 Apple Silicon 与 Intel 文件名，便于选择正确版本。

### Fixed

- 修复多来源音频转换和不同输出格式处理中的稳定性问题。
- 修复部分元数据、封面和输出文件命名在批量处理时不一致的问题。

### Breaking Changes

- 无。
