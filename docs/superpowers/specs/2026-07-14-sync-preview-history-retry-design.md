# W4DJ RKB 转换预检、历史与失败重试设计

## 目标

本次更新把转换前的预检与确认、任务历史、失败重试和错误报告整合到现有双任务同步流程中，并修复模式及无损格式选择器的重复动画和快速点击卡顿问题。

范围限定在桌面应用当前的两个同步槽位，不改变现有音频转换规则、输出格式规则和 GitHub 打包流程。

## 交互设计

### 转换前预检与汇总确认

用户点击“同时开始”后，应用先对两个已配置任务执行只读扫描，不启动转换。扫描结果以一个汇总窗口呈现，任务 1 和任务 2 各自独立展示：

- 新增文件数量：预计会创建或转换的文件
- 已存在数量：目标文件已经存在且无需处理的文件
- 将跳过数量：被当前输出策略判定为无需处理的文件
- 错误文件数量：无法读取、无法识别或无法生成目标路径的文件
- 预计输出大小：预计新增输出文件占用的空间；无法估算时显示“无法估算”
- 源目录、输出目录、当前模式和无损格式

窗口提供“取消”“返回修改”和“确认并开始转换”三个动作。取消和返回修改都不会写入历史记录。确认后才启动任务；启动使用本次扫描得到的候选集合，避免确认内容与实际处理内容不一致。扫描期间显示加载状态，防止重复发起扫描。

如果两个任务都没有可处理文件，确认按钮不可用，并提示用户检查目录或任务设置。如果存在错误文件但仍有可处理文件，允许继续启动；错误文件会进入失败列表和历史记录，界面明确提示错误数量。

### 历史记录

每个同步槽位完成、部分完成、取消或发生错误后写入一条任务记录。两个槽位由同一个批次编号关联，便于在一次“同时开始”中查看两项任务。记录保存在现有应用数据目录的独立 `history.json` 文件中，最多保留最近 50 条任务记录，超出后删除最旧记录。

每条记录包含：

- 开始时间、结束时间和耗时
- 批次编号及任务槽位
- 源目录、输出目录、模式和格式
- 预检时的新建、已存在、跳过、错误数量
- 实际完成、跳过和失败数量
- 失败文件的源路径、目标路径和错误原因
- 任务最终状态

主界面增加可折叠的“转换历史”区域。历史条目默认显示时间、状态、完成/失败数量和输出目录；展开后查看统计和失败文件。历史读取失败不影响主界面启动，应用会使用空历史并写入诊断日志。

### 失败重试和错误报告

历史条目有失败文件时显示“一键重试失败项目”和“导出错误报告”。

重试只提交该历史条目中失败的文件，不重新处理成功、已存在或跳过的文件。重试前重新检查失败文件和输出目录是否仍然存在，并再次经过同样的预检汇总确认。重试会生成一条新的历史记录，并保留原记录及其关联关系。

错误报告导出为 UTF-8 文本文件，包含任务时间、目录、模式、格式、统计以及每个失败文件的源路径、目标路径和错误原因。导出失败只提示错误，不影响任务历史。

### 选择器动画与快速点击

模式和格式按钮只在选中值真正发生变化时触发滑动动画。点击已经选中的兼容模式、无损模式、WAV 或 AIFF 不会重新添加动画类，也不会重新发起后端调用。

一次选择请求尚未返回时，相关选择按钮暂时锁定，避免快速连续点击造成多次渲染、动画重启或状态回跳。请求完成后恢复按钮；请求失败时恢复原状态并显示错误。模式切换和格式切换分别维护自己的待处理状态，互不影响。

## 技术方案

### 后端边界

在现有同步逻辑之上增加只读预检接口，复用现有的目录扫描、输出策略和冲突判断逻辑，不复制一套前端判断规则。预检返回可序列化的任务摘要、候选文件和错误项；候选文件带有源路径、目标路径、源大小和预计输出大小。

桌面控制器增加以下能力：

- 执行单个槽位或两个槽位的预检
- 根据确认后的预检结果启动同步
- 记录任务状态和失败文件
- 读取、清理和追加历史记录
- 根据历史失败项创建重试任务
- 导出错误报告

现有的暂停机制保持不变。取消、部分完成和失败都必须先完成当前文件的安全清理，再写入历史记录；临时输出文件不能被记录为成功结果。

### 数据流

```text
点击同时开始
  -> 两个任务只读预检
  -> 汇总确认窗口
  -> 用户确认
  -> 使用已确认候选集合启动同步
  -> 实时更新完成/失败状态
  -> 写入每个任务的历史记录
  -> 历史记录提供失败重试和报告导出
```

重试路径与普通启动共用预检、确认、同步和历史写入流程，区别只在于候选集合来自历史失败项。

### 前端边界

新增预检确认组件、历史记录组件和错误详情组件；主应用只负责组合视图和调用服务。预检窗口的确认按钮必须等待后端确认启动成功后才关闭，启动失败时保留窗口并显示原因。

状态更新采用不可变快照。动画触发由“旧值与新值是否不同”决定，不由每次渲染决定；选择请求期间的按钮锁定状态不写入持久化偏好。

## 错误处理

- 源目录不可读：预检任务标记为错误，确认按钮在无其他可处理任务时禁用。
- 输出目录不可写：预检显示错误并阻止该任务启动。
- FFmpeg 不可用：预检显示环境错误，避免用户确认后才发现。
- 单个文件转换失败：继续处理其他文件，并把该文件加入失败列表。
- 历史文件损坏：忽略损坏内容，写日志并从空历史继续运行。
- 重试时文件已被移动或删除：标记为新的失败项，不影响其他重试文件。
- 报告导出失败：显示可理解的错误消息，并保留历史数据。

## 测试计划

后端以公开的预检、控制器和历史接口为测试边界，前端以用户可观察的渲染和服务调用为测试边界。

必须覆盖：

- 两个任务的预检统计正确分组，预计大小可以累加，无法估算时有明确状态。
- 无可处理文件时确认按钮禁用；有错误但仍有有效文件时可以继续。
- 取消确认不会启动同步，也不会写历史。
- 完成、部分失败、取消和全失败都会写入正确的历史记录。
- 历史记录最多保留 50 条，损坏历史不会阻止启动。
- 重试只处理失败文件，并创建关联的新历史记录。
- 错误报告包含失败路径和原因，导出失败不破坏历史。
- 重复点击当前模式或当前格式不触发服务调用和动画。
- 快速点击 WAV/AIFF 时不会产生并行动画或状态回跳。
- 现有暂停、同步策略、格式转换和拖拽行为保持通过。

## 非目标

本次不加入文件夹自动监控、可配置任务数量、DJ 设备预设、BPM/调性分析、播放列表同步或云端历史同步。这些功能可以在后续版本单独规划。
